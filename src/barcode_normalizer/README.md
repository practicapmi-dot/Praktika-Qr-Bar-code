# barcode_normalizer

Нормализация (выпрямление) 1D штрих-кодов для последующего декодирования.
Классический CV на OpenCV, без нейросетей.

## Задача

На вход: полный кадр + осевой bounding box `(x1, y1, x2, y2)`, который
гарантированно вмещает один штрих-код (может захватывать фон и текст).
Код может быть повёрнут на любой угол (0–360°) и искажён перспективой
(снят под углом, на коробке).

На выходе: выпрямленное изображение, где **полосы строго вертикальны**,
перспектива устранена, код увеличен до фиксированной высоты. Готово к
подаче в декодер (pyzbar / zxing / и т.п.). Поля (quiet zones) не
добавляются.

## Установка зависимостей

```
pip install opencv-python numpy
```

## Использование

```python
import cv2
from barcode_normalizer import normalize_barcode, batch_normalize, NormalizerConfig

frame = cv2.imread("frame.jpg")
bbox = (x1, y1, x2, y2)          # осевой bounding box кода

cfg = NormalizerConfig(
    target_height=256,           # высота выпрямленного кода (апскейл)
    output_grayscale=False,      # True -> одноканальный выход
    correct_perspective=True,    # коррекция перспективы
)

# один код
norm = normalize_barcode(frame, bbox, cfg)

# несколько кодов из одного кадра
norms = batch_normalize(frame, [bbox1, bbox2, ...], cfg)

# режим отладки: вернёт (result, dbg) со всеми промежуточными шагами
result, dbg = normalize_barcode(frame, bbox, cfg, debug=True)
from barcode_normalizer.debug import render_debug
render_debug(dbg, save_path="debug.png")
```

## Конвейер (ориентация ведёт геометрию)

1. **Кроп** по bbox с небольшим padding.
2. **Предобработка** (`preprocess.py`): grayscale, мультипликативное
   вычитание фона (устранение бликов/неравномерного света), CLAHE,
   bilateral-фильтр. Бинаризации нет — при низком разрешении она вредна.
3. **Оценка угла полос** (`orientation.py`): проекционный (Radon-подобный)
   поиск угла, при котором полосы дают максимально резкий 1D-профиль.
   Отдельно разрешается неоднозначность 90° по спектральной периодичности.
4. **Поворот к вертикали**: кроп поворачивается так, что полосы становятся
   вертикальными.
5. **Сегментация зоны полос** (`segment.py`): карта «штрихкодовости»
   (когерентность структурного тензора) отделяет полосы от текста и фона;
   берётся крупнейший связный регион.
6. **Коррекция перспективы** (`perspective.py`): по разнице наклона полос
   в верхней и нижней частях зоны (истинное «веерное» схождение) строится
   трапеция → гомография в прямоугольник фиксированной высоты. Апскейл
   встроен в тот же `warpPerspective` (один ресемплинг).
7. **Выход**: цвет или grayscale, полосы вертикальны, без полей.

## Параметры (`config.py`)

Ключевые: `target_height`, `output_grayscale`, `correct_perspective`,
`perspective_min_shear`, `coherence_thresh`, `pad_ratio`, `interpolation`.

## Ограничения

- При ширине модуля < 1–2 px (очень маленькие кропы) часть информации
  физически потеряна — нормализация выпрямляет, но не восстанавливает
  утраченные переходы. Апскейл помогает декодеру, но не творит чудеса.
- Размытие движения не обрабатывается (по условию отсекается заранее).
- Перспектива корректируется по схождению полос; для экстремальных
  наклонов на плотных кодах возможен остаточный скос.
